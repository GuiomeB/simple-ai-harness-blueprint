# `docs/retro/` — historical record of post-release retrospectives

Output directory for the `/retro` workflow defined in `.agents/workflows/retro.md`.

One file per release cycle, named:

```
RETRO_CYCLE_<YYYY-MM-DD>.md
```

These files are the **historical record** of release retrospectives. The actual diffusion of each retro's one-action lands in the most actionable artefact — see the workflow's `Diffusion` block.

Keep this README; delete or move only when the directory has accumulated enough history to justify archiving.
